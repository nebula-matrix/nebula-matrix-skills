# IC验证测试点提取指南

本文档提供从IC模块功能规格文档中提取测试点的指导原则。

## 测试类型分类

### 1. 功能测试 (Functional Testing)

**定义**：验证模块的功能是否按照规格说明正确实现。

**提取要点**：
- 识别所有输入端口和输出端口
- 分析每个功能的输入输出关系
- 提取功能描述中的关键操作
- 关注状态机的状态转换
- 检查寄存器配置功能

**示例**：
- "验证模块在复位后能正常初始化"
- "验证数据输入端口能正确接收数据"
- "验证状态机从IDLE到ACTIVE的转换"

### 2. 边界测试 (Boundary Testing)

**定义**：测试模块在边界条件和极限情况下的行为。

**提取要点**：
- 数据宽度的最大值和最小值
- 地址空间的边界
- 计数器的溢出和下溢
- FIFO的满和空状态
- 时钟频率的最大最小限制

**示例**：
- "验证FIFO满时的反压机制"
- "验证计数器溢出时的回绕行为"
- "验证地址为0xFFFFFFFF时的访问"

### 3. 异常测试 (Exception Testing)

**定义**：测试模块在异常和错误情况下的处理能力。

**提取要点**：
- 非法输入组合
- 违反协议的时序
- 错误响应机制
- 错误恢复能力
- 异常状态处理

**示例**：
- "验证收到非法操作码时的响应"
- "验证时钟异常时的保护机制"
- "验证错误标志位的正确置位"

### 4. 性能测试 (Performance Testing)

**定义**：验证模块是否满足性能指标要求。

**提取要点**：
- 吞吐量要求
- 延迟指标
- 带宽利用率
- 处理速度
- 资源使用率

**示例**：
- "验证模块能达到100MHz工作频率"
- "验证数据传输延迟小于10个时钟周期"
- "验证吞吐量达到1Gbps"

### 5. 协议测试 (Protocol Testing)

**定义**：验证接口协议的正确性。

**提取要点**：
- 接口时序要求
- 握手协议
- 数据传输格式
- 协议状态转换
- 协议错误处理

**示例**：
- "验证AXI接口的握手时序符合协议"
- "验证APB接口的读写时序"
- "验证UART协议的帧格式"

## 测试优先级定义

### 高优先级 (High)

- 核心功能
- 影响系统稳定性的功能
- 高频使用的功能
- 安全相关的功能

### 中优先级 (Medium)

- 常规功能
- 典型使用场景
- 常见配置组合

### 低优先级 (Low)

- 边缘场景
- 可选功能
- 特殊配置
- 极少使用的功能

## 测试场景设计方法

### 1. 正向测试 (Positive Testing)

使用合法输入验证功能正确性。

**设计要点**：
- 覆盖所有合法输入组合
- 包含典型使用场景
- 验证正常工作流程

### 2. 负向测试 (Negative Testing)

使用非法输入验证错误处理。

**设计要点**：
- 非法输入值
- 违反协议的操作
- 错误注入
- 异常场景模拟

### 3. 状态覆盖测试

覆盖所有可能的状态和状态转换。

**设计要点**：
- 列出所有状态
- 覆盖所有状态转换路径
- 验证状态保持时间
- 测试状态并发

### 4. 参数化测试

使用参数化方法覆盖多种配置。

**设计要点**：
- 参数空间分析
- 边界值选择
- 典型值和特殊值
- 参数组合覆盖

## 常见IC模块测试要点

### 存储器模块 (Memory)

- 读写功能
- 地址解码
- 数据宽度
- 访问时序
- 功耗管理

### 处理器模块 (Processor)

- 指令执行
- 数据处理
- 异常处理
- 中断响应
- 流水线行为

### 接口模块 (Interface)

- 协议符合性
- 数据传输
- 流量控制
- 错误处理
- 时序要求

### 控制模块 (Controller)

- 状态机转换
- 控制信号生成
- 时序协调
- 仲裁机制
- 复位处理

### 计算模块 (Calculator)

- 算法正确性
- 精度要求
- 溢出处理
- 特殊值处理
- 性能指标

## 测试点提取最佳实践

1. **完整性**：确保覆盖所有功能点
2. **独立性**：每个测试点应独立可测
3. **可追溯性**：测试点应能追溯到规格说明
4. **可执行性**：测试点应明确具体可执行
5. **优先级**：合理设置测试优先级

## 测试点命名规范

推荐使用有意义的ID命名：

```
<模块缩写>_<功能域>_<类型>_<序号>
```

示例：
- `MEM_WR_FUNC_001` - 存储器写入功能测试001
- `CPU_EXCP_ERR_001` - CPU异常错误测试001
- `IFACE_AXI_PROT_001` - 接口AXI协议测试001
