# regslv 使用案例

## 案例 1：从 JSON 生成 reg_slv

使用已提取的 JSON 数据生成指定 Block 的 reg_slv Verilog 模块。

### 前置条件

```bash
gen-reg extract \
  --excel /home/alvin.xu/test_center/sync_rslt-main/2_Reg/Src/ASIC_PDT_V2R1_ppe_pa.xlsx \
  --output /tmp/example_regslv.json \
  --prefix ASIC_PDT_V2R1
```

### 命令

```bash
gen-reg regslv \
  --json /tmp/example_regslv.json \
  --bt pa \
  --output /tmp/example_regslv_out
```

### 终端输出

```
reg_slv 生成完成: /tmp/example_regslv_out/pa_reg_slv.sv
reg_slv 生成完成（JSON）: /tmp/example_regslv_out
```

## 案例 2：从 Excel 直接生成 reg_slv

跳过 JSON 中间步骤，直接从 Excel 生成。

### 命令

```bash
gen-reg regslv \
  --excel /home/alvin.xu/test_center/sync_rslt-main/2_Reg/Src/ASIC_PDT_V2R1_ppe_pa.xlsx \
  --sheet pa \
  --output /tmp/example_regslv_direct
```

### 终端输出

```
reg_slv 生成完成: /tmp/example_regslv_direct/pa_reg_slv.sv
reg_slv 生成完成（Excel）: /tmp/example_regslv_direct
```

## 案例 3：缺少必需参数

未同时提供 `--json` + `--bt` 或 `--excel` + `--sheet` 时，会报错。

### 命令

```bash
gen-reg regslv --json /tmp/example.json --output /tmp/out
```

### 终端输出

```
Error: 请指定 --excel + --sheet 或 --json + --bt
```

### 返回值

退出码 `1`

## 案例 4：从 Excel 直接生成并执行数据检查（失败）

Excel 直接生成时添加 `--check`，会在生成前先执行常规数据检查。若数据有误则报错退出，不会生成文件。

### 命令

```bash
gen-reg regslv \
  --excel /home/alvin.xu/test_center/sync_rslt-main/2_Reg/Src/ASIC_PDT_V2R1_ppe_pa.xlsx \
  --sheet pa \
  --check \
  --output /tmp/example_regslv_check_fail
```

### 终端输出

```
ERROR, module: pa reg: upa_ext_conf_table, offset_addr: 0x5000, reg_width(32) shoule be equal with all(field_bits)(28)!
```

### 返回值

退出码 `1`

---

## 案例 5：从 Excel 直接生成并执行 CIF 检查

Excel 直接生成时添加 `--cif-check --cif <file>`，会在生成前先验证寄存器地址是否落在 CIF 定义的范围内。

### 命令

```bash
gen-reg regslv \
  --excel /home/alvin.xu/test_center/sync_rslt-main/2_Reg/Src/ASIC_PDT_V2R1_ppe_pa.xlsx \
  --sheet pa \
  --cif-check \
  --cif ./ASIC_PDT_V2R1_CIF地址映射.xlsx \
  --output /tmp/example_regslv_cif
```

### 终端输出（通过时）

```
reg_slv 生成完成: /tmp/example_regslv_cif/pa_reg_slv.sv
reg_slv 生成完成（Excel）: /tmp/example_regslv_cif
```

### 返回值

退出码 `0`
