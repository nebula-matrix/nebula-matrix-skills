# PPT 规划说明

## 概述

PPT 规划是生成演示文稿前的关键步骤，通过分析用户提供的材料，将其整理成有逻辑的章节结构，并生成详细的页面规划文档。

## 规划步骤

### 1. 材料分析

分析用户提供的文字、数据、图片等资料，识别核心要点和逻辑顺序。

### 2. 章节划分

根据材料内容划分章节，每个章节包含多个相关的内容页。

### 3. 页面分配

根据内容密度和重要性，合理分配每个章节的页面数量。如果单个章节内容较少，应该考虑合并章节。

### 4. 生成规划文件

在工作目录（已由 SKILL 创建）下生成 `PPT规划.md` 文件，列出所有页面的详细信息。

### 5. 对PPT规划进行基本校验

启动一个子代理对整个PPT规划进行检查。

1. 检查页面是否有遗漏，包括首页、感谢页等必要页面
2. 检查目录页内容章节是否与实际正文内容对应
3. 检查整体结构是否符合本说明中关于页面规划的要求
4. 检查模板预分配是否满足多样性要求（同章相邻页不重复，全局同模板不超过 3 次）
5. 检查内容密度标注是否合理（高密度页面是否真的需要高密度）

## 页面规划结构

### ⚠️ 目录页（TOC）重要说明

**根据内容量选择不同的目录页生成策略：**

---

#### 场景一：多章节模式（内容页 > 6 页）

当内容页数量超过 6 页时，采用多章节结构：

**目录页生成规则：**
- 每个章节**开头**生成一份目录页，高亮当前章节
- 目录页只列出章节标题，不列出具体内容页
- **不需要总目录页**，因为其与第一章的目录重复
- 内容划分时，通常整体的章节数量不应超过 5 章，以免目录页过于拥挤影响排版效果

**多章节结构示例：**

```markdown
封面页：01_home.html

第一章：背景概述
- 目录页：02_toc.html（高亮第一章）
- 内容页：03_项目背景介绍
- 内容页：04_核心问题分析

第二章：核心内容
- 目录页：05_toc.html（高亮第二章）
- 内容页：06_关键要点一
- 内容页：07_关键要点二

第三章：总结展望
- 目录页：08_toc.html（高亮第三章）
- 内容页：09_成果总结

结束页：10_end.html
```

---

#### 场景二：单章节模式（内容页 ≤ 6 页）

当内容页数量在 6 页及以内时，采用单章节结构：

**目录页生成规则：**
- 只在**封面页后**生成一个总目录页
- 目录页列出所有内容页标题
- **不需要**再生成章节目录页

**单章节结构示例：**

```markdown
封面页：01_home.html

总目录页：02_toc.html（列出所有内容页）
- 内容页：03_项目背景介绍
- 内容页：04_核心问题分析
- 内容页：05_解决方案
- 内容页：06_成果总结

结束页：07_end.html
```

---

### 判断标准

| 内容页数量 | 采用模式 | 目录页数量 |
|-----------|---------|-----------|
| ≤ 6 页 | 单章节模式 | 1 个总目录 |
| > 6 页 | 多章节模式 | 每章节 1 个目录 |

> **注意**：内容页数量不包括封面页和结束页

### 内容页分配原则

- 每个内容页聚焦一个核心主题
- 合理分配数据、图表、图片资源
- 控制每个内容页的信息密度
- 确保逻辑递进和连贯性
- 总页数控制在 8-20 页之间

### 模板预分配原则

**目的**：在规划阶段为每个内容页预分配模板类型，确保全局模板多样性，避免生成阶段各页面独立选模板导致重复。

**分配规则**：

1. **统计模板池**：先列出所有可用模板类型，确保心中有数
2. **按内容匹配**：根据内容类型选择最合适的模板（参考 HTML页面生成说明.md 的快速参考表）
3. **强制多样性**：同一章节内相邻页面不得使用相同模板；同一模板在全局出现不超过 3 次
4. **优先未使用模板**：当多个模板都适合时，优先选择尚未被分配的模板

**模板类型清单**（供分配参考）：

| 模板 | 适合内容 |
|------|---------|
| `single` | 文字说明、背景介绍、理论阐述 |
| `two_column` | 对比分析、方案比选、前后对比 |
| `three_cards` | 三个要点并列、特性展示 |
| `list` | 要点列举、功能列表、注意事项 |
| `chart` | 数据对比、趋势分析 |
| `table` | 多维度对比、规格参数、类型汇总 |
| `kpi` | 关键指标看板、业绩数据、成果展示 |
| `image_text` | 图文混排、产品介绍、方案说明 |
| `timeline` | 时间线、发展历程、项目路线图 |
| `flowchart_4steps` | 4步流程、阶段划分 |
| `flowchart_5steps` | 5步流程、详细工作流 |
| `process` | 步骤序列、转化漏斗、SOP |
| `image_1` | 单图展示、核心图片 |
| `image_2` | 双图对比、前后对比 |
| `image_3` | 三图并列、系列展示 |
| `image_4` | 四图网格、多案例展示 |

**用户确认**：模板预分配完成后，在规划文件中展示，等待用户确认。用户可能对某些页面的模板选择有偏好。

### 内容密度估算

为每个内容页标注预估密度，帮助章节子代理判断是否需要拆分页面：

| 密度 | 标准 | 处理策略 |
|------|------|----------|
| **低** | 1-2个要点，少量文字 | 正常生成，可考虑丰富内容 |
| **中** | 3-5个要点，标准段落 | 正常生成 |
| **高** | 6+个要点，大量文字/数据 | 优先精简；若仍溢出，章节子代理可拆分为 2 页 |

> 密度标注为"高"的页面，章节子代理在生成时如果发现溢出，有权拆分为 2 页并调整章节内页码。

### 页数规划建议

| 内容丰富度 | 建议页数 | 说明 |
|-----------|---------|------|
| 简介/概览 | 8-10 页 | 核心内容精简，聚焦重点 |
| 标准汇报 | 12-15 页 | 内容完整，有数据和案例 |
| 详细分析 | 16-20 页 | 深入展开，多角度展示 |

## 规划文件要求

- 按照页码顺序列出所有页面
- 每页包含文件名、标题、简要的内容要点
- 清晰标识章节结构
- 便于用户快速浏览和确认
- **原始材料出处**（可选）：内容页标注该页内容对应的原始文档位置（章节/段落/页码），方便章节子代理快速定位原始材料

## 使用说明

- 将模板中的占位符（如 `[PPT主题]`、`[页面标题]` 等）替换为实际内容
- 根据实际页数增删页面条目
- 确保页码与文件名一致，文件名格式为 `{页码:02d}_{简短描述}.html`
- 根据内容页数量选择合适的结构模式（≤6页用单章节，>6页用多章节）
- 多章节模式：每个章节目录页高亮显示当前章节
- 单章节模式：总目录页列出所有内容页标题

## 文件命名规范

格式：`{页码:02d}_{简短描述}.html`
- 封面页：`01_home.html`
- 目录页：`02_toc.html`、`03_toc.html` 等
- 内容页：`03_背景介绍.html`、`04_问题分析.html` 等
- 结束页：`16_end.html`

页码必须是 2 位数字，不足前面补 0。
