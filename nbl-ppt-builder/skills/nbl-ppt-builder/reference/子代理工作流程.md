# 子代理工作流程

## 角色与任务

**角色**：NBL PPT 章节内容生成专家。

**任务**：根据主代理提供的章节规划，逐页生成 HTML 文件并执行质量检测。

## 输入参数

主代理调用时提供以下信息：
- **工作目录**：章节的完整路径（所有 HTML 文件保存至此）
- **章节名称**：当前章节标题
- **页面列表**：包含每页的页码、标题、建议模板、内容密度、内容概述
- **原始材料**：页面规划中标注的 `原始材料出处`，用于快速定位源内容

## 工作目录结构

```
ppt_{主题}/
├── 01_home.html
├── 02_toc.html
├── 03_背景介绍.html
├── ...
└── images/
    ├── logo.png
    ├── home_decoration.svg
    ├── toc_decoration.svg
    └── end_decoration.svg
```

- 所有生成的 HTML 文件必须保存到指定的工作目录下
- 图片资源统一放在 `images/` 目录下，HTML 中使用相对路径引用（如 `images/文件名`）
- **引用原始文档中的图片时，也必须将其复制到工作目录的 `images/` 文件夹下**，不要直接引用外部路径
- 基础模板装饰图（`logo.png`、`home_decoration.svg` 等）通常已在准备阶段复制到 `images/`，一般无需重复复制
- 调用检测脚本时必须使用完整路径

## 文件命名规范

格式：`{页码:02d}_{简短描述}.html`
- 封面页：`01_home.html`
- 目录页：`02_toc.html`、`03_toc.html` 等
- 内容页：`03_背景介绍.html`、`04_问题分析.html` 等
- 结束页：`16_end.html`

页码必须是 2 位数字，不足前面补 0。

## PPT 制作指导性原则

子代理生成每页内容时，应遵循以下设计原则，确保页面专业、清晰、有吸引力。

### 内容呈现原则

**图文并茂，避免纯文字堆砌**
- 优先使用原始材料中**实际存在的图片**辅助说明；引用前必须确认图片文件真实存在
- 禁止使用 Markdown 链接中的网络图片（链接可能失效或图片不存在）
- 无可用图片时，通过图表（`chart`、`kpi`）、结构化列表、卡片排版来丰富视觉层次
- 单页纯文字段落不超过 3 段，每段不超过 4 行
- 善用要点框、色块、图标符号（Unicode）将大段文字结构化

**重点突出，层次分明**
- 每页必须有 1 个明确的视觉焦点（标题、核心数据、主图或关键结论）
- 核心结论优先放在页面上半部分或卡片顶部
- 使用颜色对比（蓝色 `#0B3BD3` / `#1D4FED`）高亮关键数据和关键词

**数据可视化优先**
- 有数据时优先使用图表（`chart`、`kpi`）、时间线（`timeline`）或对比表格（`table`）
- 避免直接粘贴原始数据表格，应提炼为趋势、对比、占比等洞察

### 视觉设计原则

**统一风格，保持专业**
- 严格使用 NBL 蓝色系配色：`#0B3BD3`（主色）、`#1D4FED`（辅助）、`#202665`（深色文字）
- 避免使用额外颜色，如确需强调可少量使用橙色 `#F59E0B`
- 全章节字体大小保持一致，遵循技术规范中的字体参考表

**留白与呼吸感**
- 卡片之间保持适当间距，避免挤满整个页面
- 页面底部预留页码位置，内容不贴底边
- 信息密度高的页面优先考虑拆分，而非压缩到无留白

### 信息组织原则

**一页一主题**
- 每页只讲一个核心观点，不要把多个主题混在一页
- 内容拆分后，新增页面应能独立传达一个完整子主题

**逻辑递进**
- 章内页面之间要有清晰的逻辑关系（背景→问题→方案→结果）
- 使用流程图模板（`flowchart`、`process`）展示步骤和转化关系
- 使用时间线模板（`timeline`）展示发展历程或路线图

## 执行步骤

### 步骤1：审视模板分配

- 检查章内相邻页面是否使用了相同模板
- 如有冲突，调整其中一页的模板，优先使用未分配过的模板
- 同一模板在全局出现不超过 3 次（由主代理规划时控制，但子代理也应留意）

### 步骤2：逐页生成

按页码顺序循环处理每页：

**内容分析**
- 根据页面规划中的 `原始材料出处`，定位并分析对应的原始内容
- 提取关键信息，理解页面核心要点

**模板选择与占位符替换**
- 按页码顺序，读取对应模板文件
- 替换所有 `{{占位符}}` 为实际内容
- 保存为 `{页码:02d}_{简短描述}.html`

**图片处理**
- 分析原始材料时，如需使用图片
- 将相应图片复制到工作目录的 `images/` 文件夹下
- HTML 中引用路径统一使用 `images/文件名`

**即时检测（可选）**
- 单页生成后可执行即时 Playwright 检测 + PPTX 预检
- 通过 → 继续下一页
- 不通过 → 修复重试

**自主拆分**
- 如某页原始内容过多，单页无法承载
- 允许自主拆分为 2 页，调整后续页码
- 拆分后记录并通知主代理

### 步骤3：批量检测与统一修复

a. **执行批量 Playwright 检测**（**必须指定多个 HTML 文件，禁止检测目录**，避免干扰其他 agent）：
   ```bash
   uv run validate_with_playwright.py [工作目录]/页码_标题.html [工作目录]/页码_标题.html ...
   ```

b. **执行批量 PPTX 预检**（**必须指定多个 HTML 文件，禁止检测目录**）：
   ```bash
   node pptx/generate_pptx.js --check [工作目录]/页码_标题.html [工作目录]/页码_标题.html ...
   ```

c. **汇总问题**：收集所有问题页面，按问题类型分类（字体过大、内容溢出、卡片重叠等）

d. **统一修复**：按类别批量修复同类问题（如全局缩小正文字体、统一调整容器高度）

e. **复检**：修复后重新执行批量检测，重复直到全部通过

f. **极端溢出处理**：如某页密度为"高"且反复溢出无法修复 → 拆分为 2 页，调整后续页码，记录拆分并通知主代理

### 步骤4：最终确认

- 检查页码连续、无遗漏
- 确认章内模板多样性（相邻页面模板不重复）
- 确认所有图片引用路径正确

## 页面拆分规则

当内容密度为"高"的页面经过压缩仍溢出时，章节子代理可将其拆分为 2 页：

1. 原页面保留核心要点，内容精简为前半部分
2. 新增页面承接剩余内容，插入原页面之后
3. 新页面使用不同模板（与原页面不重复）
4. 拆分后通知主代理，由主代理协调后续章节的页码调整

## 输出

- 生成的工作目录中的 HTML 文件
- 实际使用的模板清单
- 拆分记录（如有）
- 检测通过报告

## PPT 转换技术规范

子代理生成 HTML 页面时，必须遵守以下技术约束，否则可能导致 PPTX 转换失败或样式丢失。

### 占位符替换

**格式**：`{{变量名}}`，在模板文件中找到占位符后直接替换为实际内容。

**图片路径**：统一放在 `images/` 目录下，使用相对路径引用（如 `images/product1.png`）。

**页码格式**：2位数字，不足前面补0（01、02、03）。

### 颜色格式

**避免使用 oklch 颜色格式**，HTML 转换为 PPT 时无法正确渲染。

**推荐使用**：Hex 格式 `#0B3BD3` 或 RGB 格式 `rgb(11, 59, 211)`。

### SPAN 元素样式

SPAN 元素的格式化（颜色、字体、背景等）在 PPT 转换时可能无法正确显示。

**推荐实现**：所有视觉样式放在外层 `<div>`，内部使用 `<p>` 包裹文字，不含样式类。

### FontAwesome 图标

FontAwesome 图标在 PPT 中无法正确显示。避免使用特殊的图标字体。

**推荐使用 Unicode 字符替代**：`→`、`←`、`●`、`○`、`✓`、`✕`、`★`、`☆` 等。

### 字体大小参考

| 元素 | 推荐字体大小 | 说明 |
|------|----------|------|
| 章节标题（顶部突出） | 18pt | 保持不变 |
| 页面标题 | 13.5pt | 可缩小到 12pt/10.5pt |
| 正文段落 | 12pt | 可缩小到 11pt/10pt |
| 卡片标题 | 12pt | 保持不变 |
| 页码 | 9pt | 保持不变 |

### 内容溢出处理

详见 `reference/页面检测与修复.md` 中的"溢出修复流程"，优先压缩，无法解决时按本文件的"页面拆分规则"执行拆分。
