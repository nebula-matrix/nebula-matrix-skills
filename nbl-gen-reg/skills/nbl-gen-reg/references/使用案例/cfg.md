# cfg 使用案例

## 案例 1：从 Excel 直接生成配置表

跳过 JSON 中间步骤，直接从 Excel 生成指定 Block 的配置表。

### 命令

```bash
gen-reg cfg \
  --excel /home/alvin.xu/test_center/sheet/ASIC_PDT_V2R1_ppe_ipro.xlsx \
  --bn ipro \
  --output /tmp/example_cfg_excel
```

### 终端输出

```
cfg 生成完成（Excel）: /tmp/example_cfg_excel
```

### 结果预览

```bash
$ ls -lh /tmp/example_cfg_excel/
-rw-rw-r-- 1 user user 2.7K May  4 12:00 ipro_reg_tbl_cfg_decl.sv
-rw-rw-r-- 1 user user  17K May  4 12:00 ipro_reg_tbl_cfg.sv
-rw-rw-r-- 1 user user 7.9K May  4 12:00 ipro_init_cfg_by_ral.sv
```

## 案例 2：从 Excel 生成并执行数据检查（失败）

Excel 直接生成时添加 `--check`，会在生成前先执行常规数据检查。若数据有误则报错退出，不会生成文件。

### 命令

```bash
gen-reg cfg \
  --excel /home/alvin.xu/test_center/sync_rslt-main/2_Reg/Src/ASIC_PDT_V2R1_ppe_pa.xlsx \
  --bn pa \
  --check \
  --output /tmp/example_cfg_check_fail
```

### 终端输出

```
ERROR, module: pa reg: upa_ext_conf_table, offset_addr: 0x5000, reg_width(32) shoule be equal with all(field_bits)(28)!
```

### 返回值

退出码 `1`

## 案例 3：从 JSON 单 bt 生成配置表

使用已提取的 JSON 数据生成指定 Block 的寄存器配置表。

### 前置条件

```bash
gen-reg extract \
  --excel /home/alvin.xu/test_center/sync_rslt-main/2_Reg/Src/ASIC_PDT_V2R1_ppe_pa.xlsx \
  --output /tmp/example_cfg.json \
  --prefix ASIC_PDT_V2R1
```

### 命令

```bash
gen-reg cfg \
  --json /tmp/example_cfg.json \
  --bt pa \
  --output /tmp/example_cfg_out
```

### 终端输出

```
cfg 生成完成（JSON+bt）: /tmp/example_cfg_out
```

### 结果预览

```bash
$ ls -lh /tmp/example_cfg_out/
total 158K
-rw-rw-r-- 1 user user 16K May  3 12:00 pa_reg_tbl_cfg_decl.sv
-rw-rw-r-- 1 user user 103K May  3 12:00 pa_reg_tbl_cfg.sv
-rw-rw-r-- 1 user user 39K May  3 12:00 pa_init_cfg_by_ral.sv
```

### 文件说明

| 文件 | 大小（约） | 说明 |
|------|----------|------|
| `pa_reg_tbl_cfg_decl.sv` | 16KB | 配置表结构体声明 |
| `pa_reg_tbl_cfg.sv` | 103KB | 配置表实例化及赋值逻辑 |
| `pa_init_cfg_by_ral.sv` | 39KB | RAL 配置表初始化代码 |

### 内容预览

```bash
$ head -15 /tmp/example_cfg_out/pa_reg_tbl_cfg_decl.sv
// ============================================================
// File: pa_reg_tbl_cfg_decl.sv
// Description: Register Table Config Declaration
// Auto-generated by gen_reg_tools
// ============================================================

`ifndef __UPA_REG_TBL_CFG_DECL_SV__
`define __UPA_REG_TBL_CFG_DECL_SV__

typedef struct packed {
    logic [31:0] field_a;
    logic [31:0] field_b;
    ...
} pa_reg_tbl_cfg_t;
```

## 案例 4：从 JSON + cfg.py 系统级生成

从 JSON 和配置脚本生成系统中所有 Block 的配置表。JSON 必须由目录批量提取，且必须包含 cfg 中 `inst_map` 定义的所有 BT。

### 前置条件

```bash
# 1. 目录批量提取 JSON（确保包含所有 BT）
gen-reg extract \
  --excel-dir /home/alvin.xu/test_center/sheet \
  --output /tmp/all.json \
  --prefix ASIC_PDT_V2R1

# 2. 准备 cfg.py（必须定义 top_name 和 inst_map）
cat > /tmp/cfg_test_cfg.py << 'EOF'
inst_map = {
    'bm': [1, 0x0050_0000],
    'ped': [1, 0x0051_0000],
}

top_name = 'datapath'
EOF
```

**配置文件说明：**

| 变量 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `inst_map` | `dict[str, list]` | 是 | 定义各 BT 的实例信息。Key 为 BT 名称（小写）；Value 为 `[count, base_addr]` 列表，其中 `count` 为该 BT 的实例数量，`base_addr` 为该 BT 的基地址 |
| `top_name` | `str` | 是 | 顶层系统名称，系统级模式下 cfg 输出文件名使用此值 |

### 命令

```bash
gen-reg cfg \
  --json /tmp/all.json \
  --cfg /tmp/cfg_test_cfg.py \
  --output /tmp/example_cfg_sys
```

### 终端输出

```
cfg 生成完成（JSON+cfg）: /tmp/example_cfg_sys
```

### 结果预览

```bash
$ ls -lh /tmp/example_cfg_sys/
total 280K
-rw-rw-r-- 1 user user  50K May  4 12:00 datapath_reg_tbl_cfg_decl.sv
-rw-rw-r-- 1 user user 120K May  4 12:00 datapath_reg_tbl_cfg.sv
-rw-rw-r--  1 user user 50K May  4 12:00 datapath_init_cfg_by_ral.sv
```

系统级模式下，所有 BT 的配置汇总到 `{top_name}_*.sv` 三个文件中：
- `datapath_reg_tbl_cfg_decl.sv` — 汇总所有 BT 的结构体声明
- `datapath_reg_tbl_cfg.sv` — 按 BT 顺序拼接所有类定义
- `datapath_init_cfg_by_ral.sv` — 按 BT 顺序拼接所有 task

## 案例 5：JSON 中缺少 cfg 指定的 BT（校验失败）

若 JSON 只包含 `bm`，但 cfg 要求 `bm` 和 `ped`，会报错退出。

### 前置条件

```bash
# 只提取了 bm
gen-reg extract \
  --excel /home/alvin.xu/test_center/sheet/ASIC_PDT_V2R1_datapath_bm.xlsx \
  --output /tmp/only_bm.json \
  --prefix ASIC_PDT_V2R1
```

### 命令

```bash
gen-reg cfg \
  --json /tmp/only_bm.json \
  --cfg /tmp/cfg_test_cfg.py \
  --output /tmp/example_cfg_fail
```

### 终端输出

```
Error: 配置文件中的 BT 在 JSON 中未找到: ped
```

### 返回值

退出码 `1`

---

## 案例 6：缺少必需参数

未提供有效的参数组合时，会报错。

### 命令

```bash
gen-reg cfg --json /tmp/example.json --output /tmp/out
```

### 终端输出

```
Error: 请指定 --excel + --bn 或 --json + --bt 或 --json + --cfg
```

### 返回值

退出码 `1`
